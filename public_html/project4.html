SpotShare

Multimedia & Location Tracking App

Video Description

Three students are interacting with the melodic ink installation

Developed individually for my IAT 359: Mobile Computing course, SpotShare is a mobile application designed to help users capture and explore popular food, scenery, and entertainment spots through multimedia sharing and location-based features.

01

Introduction

SpotShare is a mobile application designed to share experiences of food, scenery, and popular entertainment spots. The primary purpose of SpotShare is to help users easily capture and explore popular places through multimedia sharing. By integrating features like photos, text, geographic location (GPS), and weather data, the app enables users to showcase their favorite spots in an intuitive and visually engaging way.

02

Authentication Module

SpotShare integrates Firebase Authentication to manage user sign-up and login processes. This ensures that only authenticated users can access the app's features and store data (e.g., user profiles, posts)

03

APIs Integration

Google Places API:
SpotShare integrates the Google Places API to allow users to search for and select locations for their posts. This API ensures users can find accurate place names instead of dealing with raw latitude and longitude.

OpenWeather API:
SpotShare integrates OpenWeather API to fetch the current weather data based on the location of their posts. This information is displayed alongside posts to provide users with context about the weather at the location they are sharing.

04

Data Storage

Local Storage:
User login information is stored locally using AsyncStorage, ensuring that users stay logged in even when the app is reopened after being sent to the background or the device restart. This improves the app's usability by providing a login experience without requiring repeated authentication.

Structured Data:
Most of the app's structured data, such as user profiles and post information is stored in Firestore. Uploaded images are stored in Firebase Storage. The image URLs are linked back to the corresponding user data in the Firestore. With data stored in Firestore and Firebase Storage, SpotShare supports multiple users and devices, and users can easily fetch real-time data.

05

Google Maps Integration

Users can access a detailed map view showing their current location and nearby posts. Posts are displayed as customized markers, visually distinguishing them from the user's location and other icons on the map.

A customized map theme has also been applied, so it has a consistent color scheme with the app's overall interface design. User can also interact with the markers to view additional information about each post.

06

Native Features

Camera:
The app allows users to take photos directly within the app to create posts.

GPS:
Users can locate their current location on the Map Screen. SpotShare will also provide location-based recommendations by displaying nearby posts within a specific distance.

Gesture Handling:
SpotShare makes use of native touch gestures to provide a smooth user experience. For example, pinch-to-zoom is used to zoom in on images in posts. A swipe gesture is used to delete posts. Double-tap is used to like post and trigger heart animation.
